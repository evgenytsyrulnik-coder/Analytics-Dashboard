## 3. Shared Components

### 3.1 `DateRangeSelector`

- Preset buttons: "Last 7 days", "Last 30 days", "Last 90 days".
- Custom range picker with calendar popover.
- Max range: 1 year.
- Emits `{ from: string, to: string }` in ISO date format.
- Selected range persisted in URL query params (`?from=2026-01-01&to=2026-01-31`).

### 3.2 `FilterBar`

- Dropdowns for: Team (org-level views), Agent Type, Status.
- Options populated via `/api/v1/orgs/{orgId}/teams` and `/api/v1/orgs/{orgId}/agent-types`.
- Selections persisted in URL query params.
- "Clear filters" button.

### 3.3 `MetricCard`

- Displays a single KPI: label, value, optional comparison to previous period (e.g., "+12%").
- Variants: number, currency (formatted as `$24,350.12`), percentage, duration.

### 3.4 `TimeseriesChart`

- Line chart (Recharts `LineChart`).
- Supports multiple series (e.g., runs, tokens, cost on separate y-axes).
- Tooltip on hover with formatted values.
- Granularity toggle: Daily / Hourly / Weekly (when data supports it).

### 3.5 `DataTable`

- Sortable columns (client-side for small datasets, server-side for runs list).
- Column visibility toggle.
- Row click handler for navigation (e.g., click a team row to go to team dashboard).
- Loading skeleton state.

### 3.6 `BudgetGauge`

- Semicircular gauge showing budget utilization.
- Color zones: green (0â€“50%), yellow (50â€“80%), red (80â€“100%+).
- Displays `$current / $limit` label.

### 3.7 `ExportButton`

- Triggers `POST /api/v1/orgs/{orgId}/exports` with current filters.
- Shows progress indicator.
- Polls `GET /api/v1/orgs/{orgId}/exports/{exportId}` until complete.
- Opens download URL in new tab.

### 3.8 `LastUpdatedIndicator`

- Shows "Last updated: 2 minutes ago" with relative time.
- Refresh button triggers React Query invalidation.

## 5. Data Fetching Strategy

| Concern | Approach |
|---|---|
| Server state | TanStack Query with `staleTime: 5 * 60 * 1000` (5 min) matching backend freshness SLA |
| Cache invalidation | Manual invalidation on "Refresh" click; automatic refetch on window focus |
| Loading states | Skeleton loaders for cards, charts, tables |
| Error states | Inline error banner with "Retry" button; toast for transient errors |
| Optimistic updates | Budget create/update/delete use optimistic updates |
| Prefetching | Prefetch team data on org dashboard hover; prefetch run detail on runs list hover |
| URL state | Date range, filters, and sort order stored in URL search params via `useSearchParams` |

## 7. Accessibility Requirements

- All interactive elements keyboard navigable.
- ARIA labels on charts (with data table fallback for screen readers).
- Color choices meet WCAG 2.1 AA contrast ratios (4.5:1 for text, 3:1 for UI components).
- Chart color palettes distinguishable for colorblind users (verified with Coblis simulator).
- Focus indicators visible on all focusable elements.
